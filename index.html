# What
* A new type of code that browsers can run. * Powered by a new VM. * Powered by new APIs.
* It's not "to the metal" like assembly. * It's not asm.js. * It is fast (20% slower than native).
* It supplements Javascript, it does not replace it. * A new compilation target for other languages. * Still some things missing.
# Why
* It's new and interesting. * It's already supported and used. * While in flux, fairly stable.
![Wasm support](images/support.png)
* You need speed. * You have code in another language that you want to reuse. * You want to port an existing application.
* Unity, Unreal, other game engines. * Autocad, Figma. * Sourcemaps parser in Firefox.
* Ember.js core / Glimmer.js core (WIP). * Add support for WebP in browsers. * Lots of libraries ported (i.e. Bullet).
# Why not
* Still young and in flux. * Only works with ints and floats. * Single threaded.
* No ECMASscript module integration. * No GC. * Harder to debug.
# How
* Most active languages: C / C++ and Rust (LLVM). * Kotlin, FAUST, LUA, OCAML, Walt, Swift, C#, Java, Oxygene. * AssemblyScript, Walt. * Bare bones: .wat.
